How to run this HTML file using a simple Python web server:

  1. Make sure your index.html file is saved in a folder (for example, C:\Users\YourName\MyWebsite).
  2. In the same folder, create a file named server.py with the following content: 
    import http.server
import socketserver

PORT = 8000

Handler = http.server.SimpleHTTPRequestHandler

with socketserver.TCPServer(("", PORT), Handler) as httpd:
    print(f"Serving at http://localhost:{PORT}")
    httpd.serve_forever()
  3. Open a terminal or command prompt, navigate to your folder, and run: 
    python server.py
  4. Open your web browser and go to http://localhost:8000/Grade12.html

This will let you view your HTML file locally as a website.

    7. 12.5.2.1; Relate and analyze the specifications of computer components to user requirements and produce graphical products using computer aided programs

    Unit 2: Computer Software

    Benchmark 12.5.2.1

    Relate and analyze the specifications of computer components to user requirements and produce graphical products using computer aided programs

    Break Down

    • Key Components:
      • Specifications of Computer Components:
        • Understanding hardware specifications (CPU, RAM, GPU, storage).
        • Knowledge of software compatibility and performance metrics.
      • User Requirements:
        • Identifying user needs (gaming, business, graphic design).
        • Analyzing how specifications meet or exceed these needs.
      • Graphical Products:
        • Proficiency in using computer-aided design (CAD) software.
        • Skills in creating and modifying graphical representations.

    Expectations

    What Students Should Learn and Achieve:

    • Analyze and relate different computer component specifications to various user requirements.
    • Demonstrate proficiency in producing graphical products using CAD software, ensuring they meet identified specifications and user needs.

    Learning Targets

    • Specifications of Computer Components:
      Target: Analyze and compare at least three different CPU and GPU specifications based on user requirements for performance.
    • User Requirements:
      Target: Develop a user profile and justify component selection based on specific needs, detailing at least three variables (e.g., budget, performance, application).
    • Graphical Products:
      Target: Create a detailed graphical product using CAD software that reflects user needs and incorporates specified components.

    Instruction and Assessment Strategies

    • Teaching Methods:
      • Lectures/Discussions: Cover hardware specifications and user requirements.
      • Hands-On Activities: Students will assemble a mock computer build based on user profiles.
      • CAD Workshops: Guided sessions where students learn to use CAD software effectively.
    • Assessment:
      • Project: Students submit a report analyzing a specific user’s needs and recommending a computer build.
      • Presentation: Students present their findings and computer build recommendations to the class.

    Terms and Definition

    CAD - In information technology, CAD stands for Computer-Aided Design. It refers to the use of computer software to create, modify, analyze, and optimize designs. CAD is widely used in various fields such as architecture, engineering, product design, and manufacturing to develop precise drawings, 3D models, and simulations. It helps improve accuracy, efficiency, and collaboration in the design process.

    Summarize

    This standard emphasizes the connection between computer component specifications and user requirements, along with the ability to produce graphical products using CAD software. Students are expected to analyze specifications, justify component choices based on user needs, and demonstrate proficiency in graphical design. The curriculum will engage students through a mix of lectures, hands-on activities, and practical assessments to ensure they meet these learning targets.

  7. Part A: Basic Specifications
    • Identify the Processor Type: CPU or GPU?
    • Core Count / Stream Processors: How many cores or stream processors are there?
    • Clock Speed: What is the base clock speed (in GHz)?
    • Memory Type & Size: What type of memory does it use? (e.g., DDR4, GDDR6). How much memory is available? (e.g., 8GB, 12GB)
    • Power Consumption: What is the TDP (Thermal Design Power)?
  8. Part B: Intermediate Specifications
    • Architecture: What is the processor’s architecture? (e.g., Intel’s Alder Lake, AMD Ryzen)
    • Performance Benchmarks: Are there any benchmark scores provided? (e.g., PassMark, 3DMark)
    • Additional Features: Supports specific technologies? (e.g., ray tracing, DLSS, hyper-threading)
  9. Part C: Advanced Comparison
    • Performance-to-Watt Ratio: How efficient is each processor?
    • Intended Use Cases: Gaming, professional rendering, scientific computing?
    • Compatibility: Is the processor compatible with certain motherboards or systems?
    • Price and Value: Based on specs, what is the approximate value?
  10. Step 4: Analysis & Discussion
    • Compare your group's CPU and GPU specs.
    • Which has higher clock speeds? Why might that matter?
    • Which has more cores or stream processors? How does that affect multitasking or parallel processing?
    • How do memory types and sizes influence performance?
    • What features are most relevant for specific tasks?
  11. Step 5: Monitoring & Assistance
    • Walk around, ask probing questions:
      • Why do higher core counts benefit certain applications?
      • How do different architectures influence performance?
      • Can two processors have similar specs but perform differently? Why?
    • Optional: Gradually Introduce Complexity
      • Start with basic specs, then add details like cache sizes, PCIe versions, or overclocking features.
      • Encourage groups to research real-world performance comparisons.

Exercise: Computer Hardware Planning Task

Scenario: Imagine you are designing computers for three different users: a Gamer, a Scientist, and a Student. For each user, choose the most appropriate components based on their needs.

  • CPU (Processor)
  • GPU (Graphics Card)
  • Local Drive (Hard Drive or SSD)
  • Cache
  • RAM (Memory)

Your Task:
For each user, suggest the most suitable CPU, GPU, Local Drive, Cache, and RAM. Justify your choices with a brief explanation of why these components are suitable for that user.

      User: Gamer

      CPU:
      GPU:
      Local Drive:
      Cache:
      RAM:
      (Provide your choices and explain briefly why they are suitable for a gamer.)

  • Scientists often work with large data sets, simulations, or complex calculations.
  • Students need a balanced system for studying, browsing, and light multitasking.
