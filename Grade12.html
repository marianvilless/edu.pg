Lesson Plan: Excel Formulas

Section A: Introduction to Excel Formulas (70 Minutes)

Objective

Students will learn how to create and use basic, intermediate, and advanced Excel formulas to perform calculations and data analysis.

Part 1: Introduction to Excel Formulas (10 minutes)

  • What is a formula? An expression used to perform calculations automatically in Excel.
  • Why use formulas? To save time, ensure accuracy, and analyze data effectively.
  • Basic syntax: Formulas start with an equal sign =, followed by the expression (e.g., =A1+B1).

Part 2: Basic Formulas and Functions (20 minutes)

  • Simple calculations:
    • Addition: =A1+B1
    • Subtraction: =A1-B1
    • Multiplication: =A1*B1
    • Division: =A1/B1
  • Using cell references: Absolute vs. relative references ($A$1 vs. A1).
  • Common functions:
    • SUM(): Adds a range of cells. Example: =SUM(A1:A5)
    • AVERAGE(): Calculates the average. Example: =AVERAGE(B1:B5)
    • MIN(), MAX(): Find smallest/largest values.
Hands-on Activity: Simple Calculations
  1. Create a new Excel sheet and enter the following data:
    CellData
    A110
    A225
    B15
    B215
  2. Perform these calculations:
    • Addition: In C1: =A1+B1
    • Subtraction: In C2: =A2-B1
    • Multiplication: In C3: =A1*B2
    • Division: In C4: =A2/B1
Part 2: Using Cell References (Absolute vs. Relative)
  • In D1, enter 2.
  • In E1, =A1*D1 (relative reference)
  • In F1, =A1*$D$1 (absolute reference)
  • Copy E1 and F1 down to E2 and F2. Observe how references change.

Observation: Relative references adjust when copied; absolute references stay fixed.

Section B: Using Common Functions

  • Enter data in A1:A5: 12, 7, 19, 5, 14
  • In B1: =SUM(A1:A5)
  • In B2: =AVERAGE(A1:A5)
  • In B3: =MIN(A1:A5)
  • In B4: =MAX(A1:A5)

Optional Challenge: Try COUNT(A1:A5) to count numbers in the range.

Section B: Intermediate Formulas and Logical Functions (20 minutes)

  • Conditional formulas: IF() function. Example: =IF(A1>50, "Pass", "Fail")
  • Nested functions: Example: =SUM(A1:A5)/COUNT(A1:A5)
  • Text functions:
    • CONCATENATE(), LEFT(), RIGHT(), MID()
    • Example: =LEFT(B2,3) extracts first 3 characters.
Hands-on Activity: Conditional, Nested, and Text Functions
  • Enter scores in A1:A5: 45, 78, 50, 62, 30
  • In B1: =IF(A1>=50, "Pass", "Fail") (drag down to B5)
  • In C1: =AVERAGEIF(A1:A5, ">50")
  • In C2: =SUMIF(A1:A5, "<=50")
  • In D1: =IF(COUNTIF(A1:A5, ">50")=0, "No high scores", AVERAGEIF(A1:A5, ">50"))
  • Text extraction:
    • B1:B4: DeepAI, Artificial, Intelligence, Machine
    • C1: =LEFT(B1,4)
    • C2: =RIGHT(B2,4)
    • C3: =MID(B3,3,4)
    • D1: =CONCATENATE(B1, " ", B2)

Section C: Advanced Formulas and Techniques

  • VLOOKUP() and HLOOKUP(): Search for data in a table. Example: =VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
  • INDEX() and MATCH(): Flexible lookup functions.
  • Array formulas: {=SUM(A1:A10*B1:B10)} (Ctrl+Shift+Enter)
  • IFERROR(): =IFERROR(A1/B1, "Error")

Hands-on: Use VLOOKUP to fetch product prices from a table.

Wrap-up and Q&A (5 minutes)

  • Recap key points
  • Share tips for practicing formulas
  • Address student questions

Lesson Title: Data Lookup Functions in Excel: VLOOKUP(), HLOOKUP(), INDEX(), and MATCH()

Learning Objectives

  • Identify the purpose and syntax of VLOOKUP() and HLOOKUP() functions.
  • Learn how to use INDEX() and MATCH() for flexible data retrieval.
  • Apply these functions through practical exercises.

Lesson Breakdown (50 Minutes)

  1. Introduction (5 min): Discuss searching for data in large tables.
  2. VLOOKUP() and HLOOKUP() (10 min):
    • VLOOKUP: =VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
    • HLOOKUP: =HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup])
    • Demonstrate with sample dataset.
  3. Practice Exercise 1: Use VLOOKUP to find department for employee ID.
  4. INDEX() and MATCH() (10 min):
    • INDEX: =INDEX(array, row_num, [column_num])
    • MATCH: =MATCH(lookup_value, lookup_array, [match_type])
    • Combine: =INDEX(salary_range, MATCH(employee_name, name_range, 0))
  5. Practice Exercise 2: Use INDEX and MATCH to find a student's score.
  6. Wrap-up and Q&A (4 min): Recap and answer questions.
  7. Homework/Extension: Create a dataset and apply all four functions.

Summary Table of Functions

Function Search Direction Flexibility Use Case Example
VLOOKUP() Vertical Moderate Find price based on product ID
HLOOKUP() Horizontal Moderate Find data in a row based on header
INDEX() Cell position-based High Retrieve data at row/column dynamically
MATCH() Find position High Find row/column number of a value

How to run this HTML file using a simple Python web server:

  1. Make sure your index.html file is saved in a folder (for example, C:\Users\YourName\MyWebsite).

    import http.server
import socketserver

PORT = 8000

Handler = http.server.SimpleHTTPRequestHandler

with socketserver.TCPServer(("", PORT), Handler) as httpd:
    print(f"Serving at http://localhost:{PORT}")
    httpd.serve_forever()
  3. Open a terminal or command prompt, navigate to your folder, and run: 
    python server.py
  4. Open your web browser and go to http://localhost:8000/Grade12.html

    7. 12.5.2.1; Relate and analyze the specifications of computer components to user requirements and produce graphical products using computer aided programs

    Unit 2: Computer Software

    Benchmark 12.5.2.1

    Relate and analyze the specifications of computer components to user requirements and produce graphical products using computer aided programs

    Break Down

    • Key Components:
      • Specifications of Computer Components:
        • Understanding hardware specifications (CPU, RAM, GPU, storage).
        • Knowledge of software compatibility and performance metrics.
      • User Requirements:
        • Identifying user needs (gaming, business, graphic design).
        • Analyzing how specifications meet or exceed these needs.
      • Graphical Products:
        • Proficiency in using computer-aided design (CAD) software.
        • Skills in creating and modifying graphical representations.

    Expectations

    What Students Should Learn and Achieve:

    • Analyze and relate different computer component specifications to various user requirements.
    • Demonstrate proficiency in producing graphical products using CAD software, ensuring they meet identified specifications and user needs.

    Learning Targets

    • Specifications of Computer Components:
      Target: Analyze and compare at least three different CPU and GPU specifications based on user requirements for performance.
    • User Requirements:
      Target: Develop a user profile and justify component selection based on specific needs, detailing at least three variables (e.g., budget, performance, application).
    • Graphical Products:
      Target: Create a detailed graphical product using CAD software that reflects user needs and incorporates specified components.

    Instruction and Assessment Strategies

    • Teaching Methods:
      • Lectures/Discussions: Cover hardware specifications and user requirements.
      • Hands-On Activities: Students will assemble a mock computer build based on user profiles.
      • CAD Workshops: Guided sessions where students learn to use CAD software effectively.
    • Assessment:
      • Project: Students submit a report analyzing a specific user’s needs and recommending a computer build.
      • Presentation: Students present their findings and computer build recommendations to the class.

    Terms and Definition

    CAD - In information technology, CAD stands for Computer-Aided Design. It refers to the use of computer software to create, modify, analyze, and optimize designs. CAD is widely used in various fields such as architecture, engineering, product design, and manufacturing to develop precise drawings, 3D models, and simulations. It helps improve accuracy, efficiency, and collaboration in the design process.

    Summarize

    This standard emphasizes the connection between computer component specifications and user requirements, along with the ability to produce graphical products using CAD software. Students are expected to analyze specifications, justify component choices based on user needs, and demonstrate proficiency in graphical design. The curriculum will engage students through a mix of lectures, hands-on activities, and practical assessments to ensure they meet these learning targets.

  7. Part A: Basic Specifications
    • Identify the Processor Type: CPU or GPU?
    • Core Count / Stream Processors: How many cores or stream processors are there?
    • Clock Speed: What is the base clock speed (in GHz)?
    • Memory Type & Size: What type of memory does it use? (e.g., DDR4, GDDR6). How much memory is available? (e.g., 8GB, 12GB)
    • Power Consumption: What is the TDP (Thermal Design Power)?
  8. Part B: Intermediate Specifications
    • Architecture: What is the processor’s architecture? (e.g., Intel’s Alder Lake, AMD Ryzen)
    • Performance Benchmarks: Are there any benchmark scores provided? (e.g., PassMark, 3DMark)
    • Additional Features: Supports specific technologies? (e.g., ray tracing, DLSS, hyper-threading)
  9. Part C: Advanced Comparison
    • Performance-to-Watt Ratio: How efficient is each processor?
    • Intended Use Cases: Gaming, professional rendering, scientific computing?
    • Compatibility: Is the processor compatible with certain motherboards or systems?
    • Price and Value: Based on specs, what is the approximate value?
  10. Step 4: Analysis & Discussion
    • Compare your group's CPU and GPU specs.
    • Which has higher clock speeds? Why might that matter?
    • Which has more cores or stream processors? How does that affect multitasking or parallel processing?
    • How do memory types and sizes influence performance?
    • What features are most relevant for specific tasks?
  11. Step 5: Monitoring & Assistance
    • Walk around, ask probing questions:
      • Why do higher core counts benefit certain applications?
      • How do different architectures influence performance?
      • Can two processors have similar specs but perform differently? Why?
    • Optional: Gradually Introduce Complexity
      • Start with basic specs, then add details like cache sizes, PCIe versions, or overclocking features.
      • Encourage groups to research real-world performance comparisons.

Section A: Introduction to Models and Simulation

Lesson 1: Models and Simulations

Reason for learning:
Understanding models and simulations is essential because they allow us to analyze, predict, and understand complex systems and processes without the need for real-world experimentation. This knowledge is fundamental in engineering, science, and technology, helping to design better products, systems, and solutions efficiently and safely.

Objectives:

  • Define what a model and a simulation are.
  • Explain the purpose and application of models and simulations.

Questions
  1. What is a model?
  2. What is a simulation?
  3. What are the types of models and simulations?
  4. What is the purpose of models and simulations?
  5. What are some examples of models and simulations?
Key Concepts
  • What is a model? A simplified representation of a real-world object, system, or process. This can be physical (e.g., a scale model), mathematical (equations and algorithms), or graphical (diagrams).
  • What is Simulation?
    • The process of using a model to imitate the behavior of a real-world system over time.
    • Typically involves computer software to run the model and observe outcomes.
  • Types of models and simulations:
    • Physical models: Scale models of buildings, vehicles, or electronic circuits.
    • Mathematical models: Equations describing system behavior, such as electrical circuits or population growth.
    • Computer simulations: Virtual environment where models are run to analyze system performance.
  • Purpose of models and simulation:
    • To test ideas and hypotheses without physical prototypes.
    • To predict system behavior under different conditions.
    • To optimize design and improve efficiency.
    • To train personnel in a safe environment.
    • To analyze complex systems that are difficult to study directly.
  • Examples:
    • Engineering design (e.g., stress analysis of structures).
    • Electronics (circuit simulations).
    • Flight training (flight simulators).
    • Environmental science (climate modeling).
    • Business processes (financial modeling).
Activity 1.0
  1. Identify examples of models and simulations you encountered (personal experience or through media).
    What was it used for?
  2. In pairs or small groups, discuss:
    • How are a model and a simulation similar?
    • How are they different?
    • Can a model exist without a simulation?
    • Can a simulation exist without a model?

Section Summary
  • Models are simplified representations of real-world systems (e.g., physical models, diagrams, equations).
  • Simulations use models to imitate real-world processes over time, usually via computer software.
  • Both are used to analyze, predict, and understand complex systems safely and efficiently.
  • Types include physical models, mathematical models, and computer simulations.
  • They help test ideas, train users, and optimize designs without needing real-world experimentation.
  • Common examples: flight simulators, climate models, circuit simulations, and financial modeling.
  • A simulation depends on a model, but a model can exist without a simulation.

Section B: Types of Mathematical Models

Lesson 2: Mathematical Models in Simulations

Reason for Learning:
Mathematical models are essential for representing real-world systems using equations, logic, and data. They help simulate and predict system behaviors in areas like physics, finance, biology, and computing. Learning about different types of mathematical models enables students to apply these tools in problem-solving, design, and simulation scenarios across science, technology, and engineering fields.

Objectives:

  • Define what a mathematical model is.
  • Identify and describe different types of mathematical models.
  • Explain how mathematical models are used in simulations.
  • Provide examples of mathematical models in real-world applications.

Questions
  1. What is a mathematical model?
  2. What are the types of mathematical models?
  3. How are mathematical models used in simulations?
  4. Why are mathematical models important?
  5. What are real-life examples of mathematical models?
Types of Mathematical Models
  • Linear Models: Use linear equations (e.g., interest calculation).
  • Nonlinear Models: Complex behaviors (e.g., population growth).
  • Statistical Models: Use data/probability (e.g., weather prediction).
  • Algorithmic Models: Step-by-step logic (e.g., sorting algorithms).
  • Differential Models: Use calculus to model change over time (e.g., heat transfer).
Activity 2.0
  1. Identify a real-life system that could use a mathematical model.
    • What type of model would fit?
    • What variables are involved?
  2. Group Discussion:
    • Why use a mathematical model instead of a physical model?
    • Choose a model type for different scenarios (fuel efficiency, test scores, climate change).
    • Advantages of running mathematical models on a computer?

  • Mathematical models describe systems using math.
  • Types: linear, nonlinear, statistical, algorithmic, differential.
  • Used for predicting and simulating real-world systems.
  • Common in science, engineering, computing, and finance.
  • Often run on computers for speed and accuracy.
